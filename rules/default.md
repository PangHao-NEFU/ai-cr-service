# 通用代码规范（默认）

## 代码风格
- 保持代码整洁，避免无用代码和注释
- 遵循项目已有的代码风格
- 使用有意义的命名

## 命名规范
- 变量和函数名要有意义，避免 `a`, `b`, `temp` 等无意义名称
- 常量使用大写或大写下划线
- 函数名使用动词或动词短语

## 注释规范
- 复杂逻辑必须有注释
- 公共 API 必须有文档注释
- 注释要准确，与代码保持同步

## 错误处理
- 不要忽略异常和错误
- 错误信息要清晰明确
- 提供恢复或重试机制

## 函数设计
- 单一职责原则
- 函数长度不超过 50 行
- 参数数量不超过 4 个

## 代码质量
- 避免重复代码（DRY）
- 避免深层嵌套
- 提前返回，减少缩进
